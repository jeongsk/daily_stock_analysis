# 한국어(ko) 다국어 지원 추가 계획

## 목표

Web UI 인터페이스와 분석 보고서·알림 본문을 한국어(`ko`)로 지원한다. 현재 시스템은 런타임에서 중국어(`zh`, 기본)·영어(`en`)만 지원하며, 한국어는 문서(`docs/README_KO.md`, `.env.example.ko`) 수준에만 존재한다.

UI 언어를 한국어로 선택하면 **새 분석 요청에 `report_language=ko`를 자동으로 전달**하여 LLM이 한국어로 보고서를 생성하도록 연동한다.

## 핵심 결정사항 (사용자 합의 완료)

1. **단계적 3-phase 도입** — 한 번에 전부 처리하면 ~100곳의 분기 통합 + 600+ UI 키 번역 + LLM 프롬프트가 단일 PR로 검증 불가능. 각 phase를 별도 검증 단위로 분리.
2. **UI ↔ 보고서 언어 자동 연동** — UI `ko` 선택 시 분석 API 호출에 `report_language=ko` 전달. (기존 두 설정은 분리 상태였으나, 연동으로 사용자 경험 일원화)
3. **용어 매핑표 기반 번역** — 핵심 도메인 용어 매핑표를 기준으로 전체 사전을 일관되게 번역. 매핑표는 Phase 1 산출물로 사전/계획 단계에서 용어 검수 가능.
4. **집중식 언어 사전 헬퍼 도입** — 흩어진 이분법 분기(`== "en" else "中文"`)와 `_LABELS_EN`/`_LABELS_ZH` 쌍둥이 상수를 `report_language.py`의 언어별 텍스트 맵 + 조회 헬퍼로 통합. ko 확장이 한 곳에서 일관 처리되고 기존 en/zh 동작이 회귀 테스트로 검증됨.

## 현재 코드 구조 (영향 범위 조사 결과)

- 언어는 **두 개의 독립 설정**: UI 언어(`UiLanguage`, localStorage) vs 보고서 언어(`REPORT_LANGUAGE`, 서버 config).
- 백엔드는 이분법으로 전면 하드코딩: `if report_language == "en" else "中文"` 패턴이 ~100곳.
- 코어 정규화: `src/report_language.py`의 `SUPPORTED_REPORT_LANGUAGES = ("zh","en")`, `_REPORT_LANGUAGE_ALIASES`, 번역 사전 5개 + 라벨 dict 6개 + `_REPORT_LABELS`(100+ 키 zh/en 중첩).
- 프론트: `apps/dsa-web/src/i18n/uiText.ts`의 `UiLanguage = 'zh' | 'en'` + `UI_TEXT = { zh, en }`(600+ 키). 토글은 2개 언어만 전환(`language === 'zh' ? 'en' : 'zh'`).
- config 등록: `src/core/config_registry.py:2437` `REPORT_LANGUAGE` enum `["zh","en"]`; `src/config.py:2397` 파서(valid zh/en, fallback zh).

## 리스크

1. **회귀 리스크 (높음)**: ~100곳 분기 통합 시 기존 en/zh 동작이 깨질 수 있음 → Phase 1에서 모든 교체 후 기존 테스트가 동일 결과를 내는지 회귀 테스트로 방어.
2. **LLM 한국어 출력 품질 (중간)**: 한국어 프롬프트로 생성된 JSON이 스키마를 준수하지 않거나 품질이 떨어질 수 있음 → Phase 3에서 한국어 보고서 end-to-end 검증.
3. **번역 누락 (중간)**: 600+ UI 키 중 누락 → TypeScript `Record<UiLanguage, Record<UiTextKey, string>>` 타입이 컴파일 타임에 누락을 방지. 백엔드는 `SUPPORTED_REPORT_LANGUAGES` 포함 여부 + 키셋 동일성 테스트로 방어.
4. **Fallback 정책 (낮음)**: ko 사전에 키가 빠진 경우의 fallback은 `zh`(원본/가장 완전)을 기본으로 하되, UI는 타입 시스템으로 사실상 누락 불가. 백엔드 `_REPORT_LABELS`는 get 시 `KeyError`가 나지 않도록 ko 키셋을 en/zh와 1:1로 채움.
5. **연동 부작용 (낮음)**: UI↔보고서 연동으로 기존 `REPORT_LANGUAGE` 서버 기본값(zh) 사용자의 동작이 바뀌지 않도록, 연동은 "UI에서 ko 명시 선택 시에만" 적용하고 서버 기본값은 zh 유지.

## Out of Scope

- 데스크톱 일렉트론 번역 리소스 분리(Phase 2 Web 산출물을 재사용; 별도 리소스 번들링은 별도 작업)
- 추가 언어(일본어 등) — ko를 먼저 안정화
- 한국어 LLM 모델/채널 자동 선택(기존 LLM_CHANNELS 그대로 사용, 언어만 ko)
- 번역 메모리/TMS 도입(수작업 사전 유지)
