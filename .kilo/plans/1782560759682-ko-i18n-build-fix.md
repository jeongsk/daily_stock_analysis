# Plan: Complete `ko` (Korean) frontend migration to fix the Docker/web build

## Context

The Docker build fails at `[analyzer web-builder 6/6] RUN npm run build` because
`tsc -b && vite build` reports TypeScript errors. Root cause: an in-progress
Korean (`ko`) localization widened `UiLanguage` and `ReportLanguage` to include
`'ko'` and migrated several files, but **13 sites were left incomplete**.

Already migrated correctly (use as reference patterns for terminology/style):
- `src/i18n/uiText.ts` тАФ has a full `ko` block (line 1446).
- `src/locales/featureText.ts`, `src/utils/reportLanguage.ts`,
  `src/utils/marketPhase.ts`, `src/utils/uiLanguage.ts` тАФ have `ko`.

Approved approach: **author complete, natural Korean** for every `ko` gap (not
fallback to en/zh), so the Korean UI is fully functional and consistent with the
rest of the migration.

Key mechanism to remember: in `uiText.ts`, `export type UiTextKey = keyof typeof zh`
(line 723). Any key present in `en`/`ko` MUST also exist in `zh`, or the type
narrows it out and breaks both the declaration and every `t('that.key')` call.
